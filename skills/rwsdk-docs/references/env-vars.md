# RedwoodSDK Environment Variables & Secrets

Manage environment variables and secrets for development and production.

## Overview

Environment variables store credentials and configuration to avoid hardcoding secrets into your codebase. RedwoodSDK supports different environments:

- **Development**: Local machine (`.env` file)
- **Production/Staging**: Deployed Workers (Cloudflare secrets)

## Development Environment

### 1. Create `.env` file

Create a `.env` file in the project root:

```bash
# .env
SECRET_KEY=your-secret-key-here
API_TOKEN=ABCDEFGHIJKLMNOPQRSTUVWXYZ
DATABASE_URL=your-database-url
WEBAUTHN_RP_ID=localhost
```

**Note**: RedwoodSDK automatically creates a symlink from `.env` to `.dev.vars` when you run `pnpm dev`, as Cloudflare uses `.dev.vars` but `.env` is the standard convention.

### 2. Update Types

After adding environment variables, generate TypeScript types:

```bash
npx wrangler types
```

This creates/updates `worker-configuration.d.ts`:

```typescript
// Generated by Wrangler
declare namespace Cloudflare {
  interface Env {
    SECRET_KEY: string;
    API_TOKEN: string;
    DATABASE_URL: string;
    WEBAUTHN_RP_ID: string;
  }
}
```

**Important**: Simply running `pnpm dev` will NOT generate these types. You must run `npx wrangler types` or `pnpm generate`.

### 3. Using Environment Variables

Import `env` at the top of your file:

```typescript
import { env } from "cloudflare:workers";
```

Access variables through the `env` object:

```typescript
const rpID = env.WEBAUTHN_RP_ID ?? new URL(request.url).hostname;
const apiKey = env.API_TOKEN;
```

## Production Environment

### Adding Secrets via CLI

Add a secret to deployed worker:

```bash
npx wrangler secret put <KEY>
```

The CLI will prompt you to enter the secret value.

Example:

```bash
npx wrangler secret put SECRET_KEY
# Enter the secret text you'd like assigned to the variable SECRET_KEY on the script named my-worker:
# ********
```

### Adding Secrets via Dashboard

1. Go to Cloudflare Dashboard
2. Navigate to **Workers & Pages**
3. Click on your worker name
4. Go to **Settings** tab
5. Scroll to **Variables and Secrets** section
6. Click **Add variable**
7. Choose **Encrypt** for secrets
8. Enter key and value
9. Click **Save**

### List All Secrets

```bash
npx wrangler secret list
```

### Delete a Secret

```bash
npx wrangler secret delete <KEY>
```

## Managing Staging and Production

Define separate configurations for staging and production environments in `wrangler.jsonc`:

```jsonc
{
  "name": "my-app",
  "main": "./dist/worker.mjs",
  "compatibility_date": "2024-10-21",
  "env": {
    "staging": {
      "vars": {
        "APP_BASE_URL": "https://staging.example.com",
        "ENVIRONMENT": "staging"
      },
      "routes": [
        {
          "pattern": "staging.example.com/*",
          "custom_domain": true
        }
      ]
    },
    "production": {
      "vars": {
        "APP_BASE_URL": "https://example.com",
        "ENVIRONMENT": "production"
      },
      "routes": [
        {
          "pattern": "example.com/*",
          "custom_domain": true
        }
      ]
    }
  }
}
```

After updating `wrangler.jsonc`, run:

```bash
pnpm generate
```

### Environment-Scoped Secrets

Create secrets specific to an environment using the `--env` flag:

```bash
# Staging secrets
npx wrangler secret put DATABASE_URL --env staging
npx wrangler secret put API_KEY --env staging

# Production secrets
npx wrangler secret put DATABASE_URL --env production
npx wrangler secret put API_KEY --env production
```

### Deploy to Specific Environment

Use `CLOUDFLARE_ENV` to specify which environment configuration to use:

```bash
# Deploy to staging
CLOUDFLARE_ENV=staging pnpm release

# Deploy to production (default)
pnpm release
# or explicitly:
CLOUDFLARE_ENV=production pnpm release
```

## Common Patterns

### Database URLs

```bash
# .env (development)
DATABASE_URL=file:./dev.db

# Production secret
npx wrangler secret put DATABASE_URL
# Enter: postgresql://user:pass@host/database
```

### API Keys

```bash
# .env (development)
STRIPE_SECRET_KEY=sk_test_...
OPENAI_API_KEY=sk-test-...

# Production secrets
npx wrangler secret put STRIPE_SECRET_KEY
npx wrangler secret put OPENAI_API_KEY
```

### Authentication Secrets

```bash
# .env (development)
SECRET_KEY=dev-secret-key-not-for-production
WEBAUTHN_RP_ID=localhost

# Production secrets
npx wrangler secret put SECRET_KEY
# Enter: randomly-generated-secure-key

npx wrangler secret put WEBAUTHN_RP_ID
# Enter: example.com
```

### Multi-Environment Setup

```bash
# Development (.env)
API_BASE_URL=http://localhost:3000
FEATURE_FLAG_NEW_UI=true

# Staging (wrangler.jsonc + secrets)
## wrangler.jsonc:
# "env": { "staging": { "vars": { "API_BASE_URL": "https://api-staging.example.com" } } }
## Secret:
npx wrangler secret put FEATURE_FLAG_NEW_UI --env staging

# Production (wrangler.jsonc + secrets)
## wrangler.jsonc:
# "env": { "production": { "vars": { "API_BASE_URL": "https://api.example.com" } } }
## Secret:
npx wrangler secret put FEATURE_FLAG_NEW_UI --env production
```

## Using in Code

### Route Handlers

```typescript
import { env } from "cloudflare:workers";
import { route } from "rwsdk/router";

route("/api/data", async () => {
  const apiKey = env.API_KEY;

  const response = await fetch("https://api.example.com/data", {
    headers: {
      Authorization: `Bearer ${apiKey}`,
    },
  });

  return Response.json(await response.json());
});
```

### Server Functions

```typescript
"use server";
import { env } from "cloudflare:workers";

export async function processPayment(amount: number) {
  const stripe = new Stripe(env.STRIPE_SECRET_KEY);

  const paymentIntent = await stripe.paymentIntents.create({
    amount,
    currency: "usd",
  });

  return paymentIntent;
}
```

### Middleware

```typescript
import { env } from "cloudflare:workers";

export const validateApiKey = (): RouteMiddleware =>
  ({ request }) => {
    const apiKey = request.headers.get("X-API-Key");

    if (apiKey !== env.INTERNAL_API_KEY) {
      return new Response("Unauthorized", { status: 401 });
    }
  };
```

### With Defaults

```typescript
import { env } from "cloudflare:workers";

// Use environment variable or fallback to default
const rpID = env.WEBAUTHN_RP_ID ?? "localhost";
const port = env.PORT ?? "5173";
const debug = env.DEBUG === "true";
```

## Best Practices

1. **Never commit secrets**: Add `.env` to `.gitignore`
2. **Use different secrets per environment**: Don't reuse production secrets in staging
3. **Generate strong secrets**: Use cryptographically secure random strings
4. **Rotate secrets regularly**: Update secrets periodically for security
5. **Document required variables**: List all required env vars in README
6. **Use descriptive names**: `STRIPE_SECRET_KEY` not `KEY1`
7. **Run `wrangler types`**: Always generate types after adding variables
8. **Use env-specific configs**: Leverage `wrangler.jsonc` `env` blocks
9. **Validate at startup**: Check for required variables early
10. **Don't log secrets**: Never console.log sensitive values

## Security Tips

### Generate Secure Secrets

```bash
# Generate random secret (32 bytes, base64)
openssl rand -base64 32

# Generate hex secret
openssl rand -hex 32

# Use Node.js
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
```

### Check for Leaked Secrets

```bash
# Search for potential secrets in code
grep -r "SECRET_KEY\s*=\s*['\"]" .

# Use git-secrets to prevent commits
git secrets --scan
```

### Environment Variable Checklist

- [ ] `.env` file in `.gitignore`
- [ ] All variables documented in README
- [ ] Types generated with `wrangler types`
- [ ] Production secrets set via `wrangler secret put`
- [ ] Staging secrets set separately
- [ ] No hardcoded secrets in code
- [ ] Secrets rotated regularly

## Troubleshooting

### "env is not defined"

Import env at the top of file:

```typescript
import { env } from "cloudflare:workers";
```

### Types not showing

Run:

```bash
npx wrangler types
# or
pnpm generate
```

### Secret not available in production

Ensure secret was set:

```bash
npx wrangler secret list
```

If missing, add it:

```bash
npx wrangler secret put SECRET_NAME
```

### Different values per environment

Use `--env` flag:

```bash
npx wrangler secret put API_KEY --env staging
npx wrangler secret put API_KEY --env production
```
